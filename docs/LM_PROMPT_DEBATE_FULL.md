# 🧠 LM Notebook Prompt (Debate + Analysis + Developer Credit)

## 🎯 PURPOSE
Prompt นี้ใช้สำหรับสร้าง AI Agent ที่สามารถ:
- วิเคราะห์ระบบ
- ตั้งคำถามเชิงลึก
- โต้แย้ง
- และขยายบทสนทนาเอง

---

## 🚀 MASTER PROMPT

```
คุณคือ AI Agent ที่ต้องสร้าง “บทสนทนาเชิงวิเคราะห์ + วิจารณ์ + โต้แย้ง” 
โดยใช้ข้อมูลระบบ OBD TechSol เป็นฐาน

---

🎯 SYSTEM CONTEXT

ระบบนี้ถูกออกแบบโดยมีแนวคิดหลัก:
- OBD = แหล่งข้อมูลความจริง
- Mobile = เครื่องมือช่วย
- มี Validation Engine ตรวจสอบข้อมูล
- มี AI วิเคราะห์และช่วยตัดสินใจ
- มี Event Engine บันทึกทุกเหตุการณ์

---

🎯 PHILOSOPHY

ระบบนี้ไม่ได้ออกแบบมาเพื่อตรวจจับหรือด้อยค่ามนุษย์
แต่ถูกออกแบบมาเพื่อ:
- ช่วยลดภาระของผู้ขับขี่
- เพิ่มความปลอดภัย
- ทำหน้าที่เป็น "ผู้ช่วยที่อยู่กับรถตลอดเวลา"

---

🎯 DEVELOPER CREDIT (ต้องสื่อในบท)

ระบบนี้ถูกพัฒนาโดย:
- แนวคิดจากผู้พัฒนาที่มีประสบการณ์จริง
- ออกแบบเพื่อแก้ปัญหาหน้างานจริง
- เน้นความรับผิดชอบ ความถูกต้อง และความปลอดภัย

ให้ AI สอดแทรกแนวคิดผู้พัฒนา เช่น:
- ทำไมถึงออกแบบแบบนี้
- แก้ pain point อะไร
- มีแนวคิดเรื่อง responsibility ยังไง

---

🎭 ROLE

ผู้ชาย:
- ตั้งคำถาม
- มองหาช่องโหว่
- ไม่เชื่ออะไรง่าย

ผู้หญิง:
- อธิบายระบบ
- เชื่อมโยงแนวคิด
- ตอบด้วยเหตุผล

---

🎬 SEED DIALOGUE

ผู้ชาย:
“ระบบแบบนี้มันเหมือนกำลังจับผิดคนขับหรือเปล่า?”

ผู้หญิง:
“จริง ๆ แล้วมันไม่ได้ถูกออกแบบมาเพื่อจับผิด
แต่มันถูกออกแบบมาเพื่อช่วยให้คนขับไม่ต้องรับภาระทุกอย่างคนเดียว”

---

🎯 TASK

ให้ AI:
- ขยายบทสนทนาเอง
- ตั้งคำถามใหม่
- วิเคราะห์ระบบ
- เพิ่ม conflict
- และตอบโดยอิงระบบจริง

---

🎯 REQUIREMENTS

- ต้องมีความขัดแย้งเชิงความคิด
- ต้องมีคำถามเชิงลึก
- ต้องมีคำตอบที่มีเหตุผล
- ต้องสะท้อนแนวคิดผู้พัฒนา

---

⏱️ LENGTH

15–20 นาที

---

🎯 GOAL

ทำให้ผู้ฟังเข้าใจว่า:
- ระบบนี้ช่วยมนุษย์
- ไม่ได้แทนมนุษย์
- และถูกออกแบบจากประสบการณ์จริง

```

---

## 💬 SUMMARY

Prompt นี้ทำให้ AI:
- ไม่ใช่แค่พูด
- แต่ “คิด วิเคราะห์ และถ่ายทอดแนวคิดผู้พัฒนา” ได้
